# Summary

- ALTERED is a platform for strictly and ergonomically managing and utilizing human context (thoughts, files, data points) in a way that is optimized for preservation, querying, and transformation, as well producing and systemizing context-aware outputs.

# Data Model

- ALTERED operates on a thought-based, database-style model.

- All conceptual constructs evolve from the primitive Thought type.

- Each thought contains Content, which is optimally a short sentence describing one or many concise ideas.

- There is no limit to a thought's content (which can be useful for importing and working with large amounts of external text), but the concision of a thought dictates its efficiency, composability, and queryability.

- Each thought contains an Alias, which is a short display name that represents the content of the thought.

- Thought aliases are optional on a core level, however thoughts without an alias are considered Drafts. This is because many of the app's surfaces require a short identifier. We may offer the ability to AI-generate these in the future.

- Thoughts are arbitrarily-organized by Datasets, which are essentially tags that are represented by a Thought.

- Thoughts can have multiple Attributes, which are attached data points that represent a relation, value, or something else. These are still conceptually experimental.
